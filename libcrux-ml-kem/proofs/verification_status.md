# ML-KEM Verification Status

This file keeps track of the current verification status of the
modules in the ML-KEM implementation.  This version was generated by a
simple script, but we need a more precise way of visualizing the proof
status and this work is ongoing. The proofs evolve over time, and so
treat the table below as a rough guide, and ask the libcrux
maintainers if you have questions about the precise status and our
workplan towards complete proofs for the whole implementation.

Lax Checking means that the module translates to typed code in F*
which passes the F* lax checker.  Runtime Safety means that the module
has been proved to be free of panics, that it obeys all the
preconditions set by the Rust standard library (e.g. arrays are
accessed in bounds, arithmetic operations do not overflow, etc) as
well as the pre-conditions set by all the modules this module depends
on (e.g. range preconditions on inputs).  Correctness means that the
module has been formally verified for some math properties, or that
the outputs are within certain integer ranges, or that the function
is correct against a high-level specifiction of its input-output behavior.
