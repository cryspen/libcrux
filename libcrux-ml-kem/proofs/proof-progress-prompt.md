## Goal

| Crate | Branch worktree | Branch tip | Main worktree | Main tip |
|---|---|---|---|---|
| **ML-KEM** | `~/libcrux-trait-opacify` (branch `trait-opacify`) | will read | `~/libcrux-main` (branch `main`) | `70effe9c7` |
| **ML-DSA** | `~/libcrux-ml-dsa-proofs` (branch `ml-dsa-proofs`) | will read | `~/libcrux-main` | `70effe9c7` |
| **SHA-3** | `~/libcrux-sha3-focused` (branch `sha3-proofs-focused`) | will read | `~/libcrux-main` | `70effe9c7` |

Per-crate src-root paths inside each worktree:

| Crate | Path within branch worktree | Path within main worktree |
|---|---|---|
| ML-KEM | `libcrux-ml-kem/` | `libcrux-ml-kem/` |

## Tools you already have

Each branch worktree's `<crate-root>/proofs/` directory contains:

- **`generate_verification_status.py`** — generalized per-crate verification-status
  script. Reads a JSON config; classifies each Rust function into one of six
  tiers based on annotations + Makefile + extraction-dir contents:
  `lax / unverified / panic_free / math / bounds / hacspec`. Supports a
  `--diff PREV CURR` mode that compares two existing reports and emits per-row
  + aggregate deltas.
- **`verification_status.config.json`** — per-crate config (modules, paths,
  spec/range patterns, makefile location, F\* extraction dir).
- **`verification_status.md`** — current snapshot of the branch.
- **`proof_milestones.md`** — hand-curated milestones (NTT / Encoding / API).

The script is the same source file in all three crates (cross-compatible).

## Definitions

- **Specifications strengthened** = number of fns whose proof tier on the
  branch is **higher** than on main, where the tier order is:
  `unverified < lax < panic_free < math < bounds < hacspec`.
  In aggregate: `(panic_safe + correct count on branch) - (same on main)`,
  measured at each tier independently for granularity.
- **Panic-freedom proofs completed** = `(panic-safe count on branch) -
  (panic-safe count on main)` where panic-safe = `pf + math + bounds + hacspec`.
  Captures fns that moved from `lax`/`unverified` into ANY verified tier.
- **Correctness proofs accomplished** = sum across the three correctness
  tiers `math + bounds + hacspec`, broken down by tier so we can see if
  the gains are bound-correctness or full hacspec equivalence.
- **F\* lines written** = `git diff --stat main..HEAD -- '*.fst' '*.fsti'`
  on each branch worktree — captures both extracted files (auto-generated by
  hax) AND hand-written specs/bridge lemmas. Report additions, deletions,
  net.
- **Hand-written F\* lines** (a subset of the above) = same diff but
  restricted to non-extracted directories: `specs/`, `fstar-helpers/`,
  `proofs/fstar/extraction-patches/`, `proofs/fstar/equivalence/`,
  and any `*.fst.checked` or hand-edited extraction files.

## Steps

1. **Verify worktree state** — for each branch, `git -C <worktree> status
   --short` should be clean (or only have proof-status-related untracked
   files). Note the branch tips.

2. **Run the script on each branch** (current state) — the
   `verification_status.md` files in each worktree should already be
   up-to-date as of the last commit, but regenerate to be safe:
   ```bash
   cd ~/libcrux-trait-opacify/libcrux-ml-kem && \
     python3 proofs/generate_verification_status.py
   cd ~/libcrux-ml-dsa-proofs/libcrux-ml-dsa && \
     python3 proofs/generate_verification_status.py
   cd ~/libcrux-sha3-focused/crates/algorithms/sha3 && \
     python3 proofs/generate_verification_status.py
   ```

3. **Run the script on `main` for each crate** — use each branch's config
   to make sure the module groupings match (so the diff aligns by `(category,
   display)`):
   ```bash
   python3 ~/libcrux-trait-opacify/libcrux-ml-kem/proofs/generate_verification_status.py \
     --root ~/libcrux-main/libcrux-ml-kem \
     --config ~/libcrux-trait-opacify/libcrux-ml-kem/proofs/verification_status.config.json \
     --output /tmp/main-mlkem.md

   python3 ~/libcrux-ml-dsa-proofs/libcrux-ml-dsa/proofs/generate_verification_status.py \
     --root ~/libcrux-main/libcrux-ml-dsa \
     --config ~/libcrux-ml-dsa-proofs/libcrux-ml-dsa/proofs/verification_status.config.json \
     --output /tmp/main-mldsa.md

   python3 ~/libcrux-sha3-focused/crates/algorithms/sha3/proofs/generate_verification_status.py \
     --root ~/libcrux-main/crates/algorithms/sha3 \
     --config ~/libcrux-sha3-focused/crates/algorithms/sha3/proofs/verification_status.config.json \
     --output /tmp/main-sha3.md
   ```

4. **Diff each pair**:
   ```bash
   python3 ~/libcrux-trait-opacify/libcrux-ml-kem/proofs/generate_verification_status.py \
     --diff /tmp/main-mlkem.md \
            ~/libcrux-trait-opacify/libcrux-ml-kem/proofs/verification_status.md \
     --diff-label-prev main --diff-label-curr trait-opacify \
     --output /tmp/diff-mlkem.md

   # similarly for ml-dsa and sha3
   ```
   Each diff's `## Aggregate` section gives the per-tier deltas.

5. **Count F\* line changes** for each branch:
   ```bash
   cd ~/libcrux-trait-opacify && \
     git diff --stat main..HEAD -- '*.fst' '*.fsti' | tail -1
   cd ~/libcrux-ml-dsa-proofs && \
     git diff --stat main..HEAD -- '*.fst' '*.fsti' | tail -1
   cd ~/libcrux-sha3-focused && \
     git diff --stat main..HEAD -- '*.fst' '*.fsti' | tail -1
   ```
   Capture additions, deletions, files-changed.

6. **Count hand-written F\* lines** (specs/bridge lemmas, separate from
   auto-extracted code):
   ```bash
   cd ~/libcrux-trait-opacify && git diff --stat main..HEAD -- \
     'specs/**/*.fst' 'specs/**/*.fsti' \
     'fstar-helpers/**/*.fst' 'fstar-helpers/**/*.fsti' \
     'libcrux-ml-kem/proofs/fstar/extraction-patches/' | tail -1

   cd ~/libcrux-ml-dsa-proofs && git diff --stat main..HEAD -- \
     'specs/**/*.fst' 'specs/**/*.fsti' \
     'fstar-helpers/**/*.fst' 'fstar-helpers/**/*.fsti' \
     'libcrux-ml-dsa/proofs/fstar/extraction-patches/' | tail -1

   cd ~/libcrux-sha3-focused && git diff --stat main..HEAD -- \
     'specs/**/*.fst' 'specs/**/*.fsti' \
     'fstar-helpers/**/*.fst' 'fstar-helpers/**/*.fsti' \
     'crates/algorithms/sha3/proofs/fstar/equivalence/**/*.fst' \
     'crates/algorithms/sha3/proofs/fstar/equivalence/**/*.fsti' \
     'crates/algorithms/sha3/proofs/fstar/extraction-patches/' | tail -1
   ```
   sha3 has its functional-correctness equivalence proofs at
   `crates/algorithms/sha3/proofs/fstar/equivalence/` (NOT under
   `extraction/`); include that path. ml-kem and ml-dsa hand-written
   bridge lemmas live under `specs/<crate>/proofs/fstar/commute/`.

7. **Count Rust ensures changes** (the source of new specs, before extraction):
   ```bash
   cd ~/libcrux-trait-opacify && git diff --stat main..HEAD -- \
     'libcrux-ml-kem/src/**/*.rs' | tail -1
   # similarly for the other two
   ```
   This is informational — the spec strength comes through to F\* via
   extraction.

## Things to watch for / known quirks

- **ml-dsa Makefile** uses an *inverted* admit pattern: it lists
  `VERIFIED_MODULES`, and `ADMIT_MODULES = filter-out VERIFIED ...`. The
  script handles this automatically (via `_parse_makefile_var`); just confirm
  the Lax counts look reasonable.
- **Name mangling**: hax appends `_` to F\* segments whose Rust identifier
  ends in a digit (`ml_dsa_44` → `Libcrux_ml_dsa.Ml_dsa_44_.fst`). The
  script handles both directions.
- **Ancestor coverage**: a Rust file like `src/hash_functions.rs` may have
  no direct `Libcrux_<crate>.Hash_functions.fst` but multiple nested
  `Libcrux_<crate>.Hash_functions.{Portable,Neon,...}.fst`. The script
  marks the parent as covered.
- **lib.rs** is filtered out of F\* extraction in all three crates and
  shows up as Unverified.
- **sha3 keccakf1600 caveat**: the `keccakf1600` function has no `ensures`
  on itself; equivalence to `Hacspec_sha3.Keccak_f.keccak_f` is proven via
  separate lemmas at `proofs/fstar/equivalence/EquivImplSpec.Keccakf.*.fst`.
  The script can only see function-level ensures, so the "Hacspec" tier
  on sha3 undercounts the real proof state. Note this in the per-crate
  paragraph.
- **Diff parsing edge case**: the script's `parse_status_md` requires a
  10-column table (cat | display | mods | fns | lax | unv | pf | math |
  bounds | hacspec). If the source markdown was generated with an older
  version (different column count), regenerate first.
- **Lines counting tip**: `git diff --stat main..HEAD -- '*.fst' '*.fsti'`'s
  last line is `N files changed, X insertions(+), Y deletions(-)`. Parse
  this with `awk '{print}' | tail -1` and pull X / Y. If main lacks some
  paths entirely (e.g. `crates/algorithms/sha3/proofs/fstar/equivalence/`
  may not exist on main), git records all those as additions — that's
  the right accounting.

- The "Specs strengthened" / "Panic-freedom proofs completed" /
  "Correctness proofs" rows MUST agree with each other (the panic-safe

- For each crate, write 3-5 lines of interpretation grounded in the

- Save to `/tmp/proof-progress-summary.md` and print.
