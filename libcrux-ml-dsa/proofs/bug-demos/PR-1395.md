# PR 1395 — AVX2 `reduce` skipped 28 of 32 SIMD lanes

| Field | Value |
| --- | --- |
| Production PR | https://github.com/cryspen/libcrux/pull/1395 |
| Bug class | Functional spec — trait postcondition violation |
| FIPS reference | FIPS 204 §2.4.3 (centred Barrett reduction modulo q = 8 380 417) |
| Production location | `libcrux-ml-dsa/src/simd/avx2.rs::Operations::reduce` (`reduce_with_proof` helper, lines 169‑188) |
| F\* module | `Libcrux_ml_dsa.Simd.Avx2.reduce_with_proof` |

## What the bug looked like

The AVX2 `reduce` is required by the trait
`Libcrux_ml_dsa.Simd.Traits.Operations::reduce` to centre‑reduce **every**
one of the 32 SIMD lanes that pack a degree‑256 ML‑DSA polynomial.
Pre‑PR‑1395, the AVX2 implementation reduced only **4 of the 32** lanes
(indices 0, 8, 16, 24); the other 28 lanes kept their pre‑reduction
value (up to ≈ 2³¹ in absolute value). Downstream code that relied on
the post `|coefficient| ≤ q − 1` saw silent overflow.

```rust
// Buggy production code (PR 1395 reverted)
arithmetic::reduce(&mut simd_units[0].value);
arithmetic::reduce(&mut simd_units[8].value);
arithmetic::reduce(&mut simd_units[16].value);
arithmetic::reduce(&mut simd_units[24].value);
```

## Fix shipped in PR 1395

Replace the four explicit calls with a 32‑lane loop:

```rust
// Fixed (current production)
for i in 0..simd_units.len() {
    hax_lib::loop_invariant!(|i: usize| fstar!(r#"
        v $i <= 32 /\
        (forall (j:nat{j < 32}). j < v $i ==>
            (forall (k:nat{k < 8}).
                Spec.Intrinsics.to_i32x8 (Seq.index ${simd_units} j).f_value (mk_u64 k) ==
                Spec.MLDSA.Math.barrett_red
                    (Spec.Intrinsics.to_i32x8 (Seq.index ${_orig} j).f_value (mk_u64 k)))) /\
        (forall (j:nat{j < 32}). j >= v $i ==>
            Seq.index ${simd_units} j == Seq.index ${_orig} j)"#));
    arithmetic::reduce(&mut simd_units[i].value);
}
```

## Why this verification approach catches the bug

The FIPS‑mandated post `forall j<32. is_i32b_array_opaque(FIELD_MAX) …`
is part of the `Operations::reduce` trait contract. F\* enforces this
post on every implementation; the AVX2 impl discharges it via the
loop's invariant plus a per‑lane Barrett‑bound lemma. Drop a single
lane from the loop and the post collapses — F\* refuses to type‑check
the body. This is a structural, not heuristic, guarantee: the bug class
"forgot to reduce a SIMD lane" is impossible in the verified codebase.
